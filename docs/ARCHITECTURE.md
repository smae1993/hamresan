# معماری همرسان

## مرزهای سیستم

همرسان در نسخهٔ فعلی یک برنامهٔ کاملاً محلی است. سرور مرکزی، حساب کاربری یا API اینترنتی ندارد. هر نصب یک `deviceId` تصادفی و پایدار دارد و دستگاه‌ها فقط روی LAN یکدیگر را می‌بینند.

## لایه‌ها

### Presentation

صفحه‌ها و State Machine انتقال در `features/*/presentation` قرار دارند. UI فقط مقادیر نمایشی مانند متن فارسی حجم را می‌سازد؛ اندازه، شناسه و وضعیت در Domain با نوع واقعی نگهداری می‌شوند.

### Domain

- `Device`: دستگاه کشف‌شده و نسخهٔ پروتکل آن
- `Identity`: هویت پایدار نصب جاری
- `ContentItem`: فایل با مسیر محلی و اندازهٔ دقیق بایت
- `IncomingRequest`: پیشنهاد انتقال دریافت‌شده
- `TransferSession`: نشست ارسال یا دریافت
- `TransferProgress`: فاز، بایت‌های منتقل‌شده، سرعت و فایل جاری

### Data and transport

- `LanService`: Discovery، پیام‌های کنترلی UDP، مجوز انتقال و جریان TCP
- `TransferRepository`: مرز میان State Machine و Transport
- `ContentRepository`: اسکن محدود فایل‌های محلی و File Picker بومی
- Repositoryهای SharedPreferences: تنظیمات سبک و هویت

## چرخهٔ انتقال

1. فرستنده دستگاه را با `deviceId` انتخاب می‌کند.
2. Metadata فایل‌ها با `transferId` از طریق UDP پیشنهاد می‌شود.
3. گیرنده درخواست را قبول یا رد می‌کند.
4. در صورت قبول، گیرنده Token تصادفی و پورت TCP را برمی‌گرداند.
5. فرستنده Header نسخه‌دار و بایت‌های واقعی فایل را ارسال می‌کند.
6. گیرنده فایل را ابتدا با پسوند موقت ذخیره می‌کند.
7. SHA-256 فرستنده و گیرنده مقایسه می‌شود.
8. فایل معتبر Atomically Rename و سپس ACK نهایی ارسال می‌شود.

## قواعد معماری

- از نام دستگاه به‌عنوان شناسه استفاده نکنید.
- حجم فایل در Domain همیشه `int` و بر حسب بایت است.
- متن فرمت‌شده فقط در Presentation ساخته می‌شود.
- هیچ خطای شبکه‌ای نباید به موفقیت تبدیل شود.
- تغییر Wire Format نیازمند افزایش نسخهٔ پروتکل و تست سازگاری است.
- مسیر یا نام دریافتی از شبکه قبل از استفاده در فایل‌سیستم باید پاک‌سازی شود.
